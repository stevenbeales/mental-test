    class Minitest::Test

    Subclass Test to create your own tests. Typically you'll want a Test subclass per implementation class.

    See Minitest::Assertions

    Contest adds teardown, test and context as class methods, and the instance methods setup and teardown now iterate on the corresponding blocks. Note that all setup and teardown blocks must be defined with the block syntax. Adding setup or teardown instance methods defeats the purpose of this library.

def self.context(name, &block)
  subclass = Class.new(self)
  remove_tests(subclass)
  subclass.class_eval(&block) if block_given?
  const_set(context_name(name), subclass)
end

    runnable_methods() click to toggle source

    Returns all instance methods starting with “test_”. Based on test_order, the methods are either sorted, randomized (default), or run in parallel.

    # File vendor/bundle/ruby/2.4.0/gems/minitest-5.11.3/lib/minitest/test.rb, line 65
def self.runnable_methods
  methods = methods_matching(/^test_/)

  case self.test_order
  when :random, :parallel then
    max = methods.size
    methods.sort.sort_by { rand max }
  when :alpha, :sorted then
    methods.sort
  else
    raise "Unknown test_order: #{self.test_order.inspect}"
  end
end

def run
  with_info_handler do
    time_it do
      capture_exceptions do
        before_setup; setup; after_setup

        self.send self.name
      end

      TEARDOWN_METHODS.each do |hook|
        capture_exceptions do
          self.send hook
        end
      end
    end
  end

  Result.from self # per contract
end
